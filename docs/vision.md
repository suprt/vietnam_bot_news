## Видение проекта

Проект собирает новости из вьетнамских онлайн-изданий, автоматически категоризирует их, делает краткие русскоязычные резюме и публикует подборку дня в Telegram-канале. Целевая аудитория — русскоязычные пользователи, которым важен регулярный, сжатый и нейтральный обзор вьетнамской повестки.

### MVP-границы

- Источники: 5–10 крупнейших вьетнамских новостных сайтов, преимущественно через RSS, при необходимости — HTML-скрейпинг.
- Язык ввода: вьетнамский (возможны отдельные англоязычные ленты у некоторых источников).
- Язык вывода: русский.
- Вывод: один Telegram-канал.
- Частота: один запуск в день через GitHub Actions.
- Объём: около 100 исходных новостей в день, после фильтрации — до 5 лучших новостей в каждой категории.
- Формат в Telegram:
  - заголовок с гиперссылкой на оригинальную новость;
  - 1–2 предложения русского summary под заголовком;
  - группировка новостей по категориям внутри одного или нескольких сообщений.

### Ключевые ограничения

- Минимальная инфраструктура:
  - без выделенных БД и серверов, всё крутится в GitHub Actions;
  - состояние (отправленные новости) хранится в лёгком файловом виде (например, JSON) внутри репозитория или аналогичном простом хранилище.
- Использование Gemini Flash:
  - соблюдение лимитов бесплатного плана:
    - RPM (Requests Per Minute): 5 запросов в минуту (обновлено Google в 2025 году, было 10)
    - TPM (Tokens Per Minute): 250,000 токенов в минуту
    - RPD (Requests Per Day): 20 запросов в день (обновлено Google в 2025 году, было 250)
  - промпты должны быть компактными и давать предсказуемый формат ответа;
  - агрессивная оптимизация батчей для минимизации количества запросов.
- Надёжность:
  - стремимся не «пропускать дни»: при ошибках должны быть ретраи и понятный фолбэк;
  - логирование в GitHub Actions и, по возможности, уведомление в отдельный Telegram-чат при критических ошибках.

### Дальнейшее развитие

- Добавление других стран и языков.
- Гибкая настройка категорий и источников.
- Возможность выбора «режимов» (например, только экономика, только технологии).
- Потенциальное расширение витрины (web-архив, рассылки и т.п.), если MVP будет успешным.
